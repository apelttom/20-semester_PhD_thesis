Sonar Range Finder Block

Driver block ultrasonic range finder modules.

Library

Real-Time toolbox for 'Freescale mc9s12' based targets (rtmc9s12-Target)

Description


This block services ultrasonic range finder modules such as the Devantech SRF05; these modules are often referred to as 'SONAR' (SOund NAvigation And Ranging) modules - hence the name of this block. These modules typically have an input line as well as an output line. The input line is used to trigger a measurement. A short burst of an ultrasonic pulse train (often 40 kHz) is emitted. The receipt of the echo is then signalled through the output line of the module. Seen from the side of the Sonar Driver Block, the Output port connects to the input line of the module and the Input port is used for echo detection. The time between triggering the pulse and the receipt of the echo can be used to determine the distance of the echo producing object. This time-of-flight measurement is not always very accurate. At every sample hit, the block produces a signal which, in theory, should be proportional to the object distance. This value needs to be scaled using a gain block.

Dialog Box


Sample time
Sample time of the block. The Sample time (in seconds) defines the rate at which the block is visited by the real-time scheduler of the target. The value of the Sample time has to be an integer multiple of the base sample time of the model. The shortest achievable sample time can be determined using the display timing signals option. This option is described in the manual of the toolbox in section 2.3 Code generation options.

Output port (trigger echo pulse train)
This is the port to which the trigger input of the ultrasonic range finder module is connected.

Output pin
The chosen pin in the above Output port.

Input port (detect 'echo received')
This is the port on which the ultrasonic range finder module signals the detection of the echo.

Input pin
The chosen pin in the above Input port.
