Embedded Target for Freescale 9S12 microcontrollers
    
Freeport Communication Block, Reception

Driver block for the Reception of 'freeport' telegrams through the selected serial port of the 9S12

Library

Real-Time toolbox for 'Freescale mc9s12' based targets (rtmc9s12-Target)

Description


The 'freeport' communications system allows the exchange of short messages (up to 50 bytes; this value can be redefined to allow for larger values) through an unused serial port on the 9S12 (a 'free' port). Free ports are those which are not used by the optional External Mode interface to upload log data to MATLAB/Simulink. If the External Mode interface is not used, both ports of the 9S12 (SCI0, SCI1) can be used as freeports. They can thus be used to communicate with a further microcontroller, with a host PC or to interface to an 'intelligent sensor' with an asynchronous serial communication interface. If the External Mode interface is active, the only available freeport is the one which is not used to communicate with MATLAB/Simulink.

Dialog Box


Sample time
Sample time of the block. The Sample time (in seconds) defines the rate at which the block is visited by the real-time scheduler of the target. The value of the Sample time has to be an integer multiple of the base sample time of the model. The shortest achievable sample time can be determined using the display timing signals option. This option is described in the manual of the toolbox in section 2.3 Code generation options.
Communication port
Available ports are SCI0 and SCI1 (block used on the target) as well as COM1 - COM4 (host block).
Baudrate
The bit rate at which communication is established. Valid values range from 150 bps to 115200 bps.
Channel number
A total number of 5 channels (configurable) can be opened for freeport communication.
Number of elements
This is the number of elements of the chosen data type which are expected on the chosen channel.
Data Type
The data type can be SINGLE, INT8, UINT8, INT16, UINT16, INT32, UINT32 and BOOLEAN .
Block receives raw data
When selected, the block receives 'raw' data, i. e. no additional information such as the logical channel number etc. is expected. This is useful when communicating with serial devices other than another freeport comms blocks, e.g. an 'intelligent' sensor such as an Inertial Measurement Unit (IMU).
Block output is 'double'
Selecting this option causes received values to be converted to double precision numbers. This is useful when a subsequent display or scope block is used to visualize the data.
