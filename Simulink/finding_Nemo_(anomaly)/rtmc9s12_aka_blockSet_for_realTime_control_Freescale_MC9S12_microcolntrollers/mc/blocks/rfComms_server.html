Embedded Target for Freescale 9S12 microcontrollers
    
RF Communication Module, Server

Driver block for a Nordic nRF2401 radio module in server mode.

Library

Real-Time toolbox for 'Freescale mc9s12' based targets (rtmc9s12-Target)

Description


The nRF2401 radio module provides a cost effective solution wherever reliable wireless communication at (measured) net data rates of around 10 kbps is required. This is more than enough for many small robotics projects. A suitable circuit, the MiRF v2 is sold by SparkFun at around US$20 (August, 2007). The nRF2401 uses a server-client protocol which allows simple star-shaped communication networks to be built. Such a network always includes one server and between 1 and 5 clients. Each client connects to the server on the chosen radio channel (RF channel, 0 - 127); the frequencies of this channel is FM modulated around a central carrier frequency of 2.4 GHz. It is worth noting that all clients of a network use the same RF channel. This makes it possible to operate more than one networks in one and the same environment. As an example consider a classroom scenario in which, say, 10 groups play robotic soccer with small mobile robots. Each group would be assigned two RF channels, enough to run two separate data networks for two teams of up to 5 robots. Each group could set up a central PC with a machine vision system, providing high-level co-ordinate information to both teams. This PC would connect to two nRF2401 units, both running as servers. The mobile robots of each team would be the clients within their respective network. As there are a total of 128 RF channels, it would, in theory, be possible to have the insane number of 64 simultaneous soccer robot matches... but who would want to watch this?! :)

Dialog Box


Sample time
Sample time of the block. The Sample time (in seconds) defines the rate at which the block is visited by the real-time scheduler of the target. The value of the Sample time has to be an integer multiple of the base sample time of the model. The shortest achievable sample time can be determined using the display timing signals option. This option is described in the manual of the toolbox in section 2.3 Code generation options.
Communication mode (TX, RX)
This sets the block up to act as transmitter (TX) or as receiver (RX).
RF channel number (0 - 127)
This parameter defines the chosen radio frequency channel. Each data network has its own individual RF frequency. Up to 128 separate data networks can be operated simultaneously.
Client number
Each data network consists of 1 server and at least 1 but up to 5 clients. The clients can be set up as transmitters or receivers.
Server address
A unique 4-byte code which is used by the low-level protocol. This value will rarely need to be changed.
Client address
A unique 4-byte code which is used by the low-level protocol. This value will rarely need to be changed.
User channel number
This value specifies a logical channel number associated with a particular server-client communication channel. This means that it is possible to use one and the same RF link for more than one separate communication links. The protocol used with this system is the same as with the freeport blocks.
Number of elements to be sent/received
This value specifies how many elements of the chosen data type will be sent or received with each transmission.
Data Type
The data type can be SINGLE, INT8, UINT8, INT16, UINT16, INT32, UINT32 and BOOLEAN .
Block sends/receives raw data
When selected, the block sends/receives 'raw' data, i. e. no additional information such as the logical channel number etc. is transmitted. This should be the default setting when an application does not require more than one channel per client-server link.
